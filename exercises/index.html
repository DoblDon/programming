Introduction

This is the practical task assignment for the Interaction Programming Block Course (introductory slides). The block course is taught at the Institute of Ergonomics of the Technische Universität München in the context of master courses in Human Factors Engineering, Mechanical Engineering, and Computer Sciences in the summer and winter term. This document is authored by Markus Zimmermann, David Schopf, Niklas Lütteken, and Annika Stumpf.

The exercises aim to learn the web technologies HTML5/CSS3/JavaScript by developing a weather application during the course.

If you follow the task assignments, you will create a weather app looking similar to the image below:

D Style and Layout

Global Stylesheet (D1)

Add the same external stylesheet main.css to all .html files. Always try to use appropriate CSS selectors that only affect those elements, which are supposed to be affected. Too imprecise selectors may cause disarrangement of your elements. If necessary, improve your html structure by adding new class or id attributes. Apply the generic rule:

  • Assign ids for content which is present once (menu, current temperature, current weather, …)
  • Assign classes for elements, which are present multiple times (e.g. multiple rows of detail information, multiple buttons, …)

Layout your Views (D2/D3)

Transform your app to into a single page application

  • Instead of using multiple HTML files (one for each view) we will now use only one page for all views. Copy and paste your detailview into your index.html just below mainview. Now detailview should overlap mainview.
  • Use the display property for showing and hiding sidebar and detailview. Try the developer console to test very quickly: Add and change CSS properties directly in Chrome (Elements > Styles).

Main view

  • Use fixed heights for the mainview element (e.g. 600px), header, datepicker (both e.g. 40px) and footer (60px). Apply a width of 100% to all elements.
  • Use relative positioning for all buttons and absolute positioning for the headlines inside header and datepicker.
  • Use a fixed width and height and absolute positioning for buttons in header and datepicker.
  • Use relative positioning for all elements inside content. Set appropriate widths and center all elements or rather group of elements.
  • Use absolute positioning to pin the footer to the bottom. Set a fixed height and full width to the footer button.

Detail view

  • Use same heights and widths for the detailview elements as used before for the mainview. Set a fixed height for the content (height detailview – height header) and make it scrollable by using the property overflow: auto.
  • Use absolute positioning for the header to pin it to the top.
  • Set a fixed height for all table rows and divide the table rows into columns using percental widths.

Sidebar

  • Use same heights for sidebar as used before.
  • Use a width of 70% for the sidebar and a dark background-color.

Add style information (D4)

MainView

  • Add a dark, half-transparent background to your buttons, datepicker and footer. Use rgba to decrease the opacity of your color.
  • Create : hover and : active styles for all buttons to highlight the user interaction.
  • Use font properties to style color and font - size of the icons and appearance of headlines.
  • Use padding & margin to set enough space between the elements.

DetailView

  • Use the CSS selector :nth-child() to set a half-transparent darker background-color to odd rows and lighter background-color to even rows.
  • If the button classes where applied correctly in the HTML part, the close button and header should be already styled.

Sidebar

  • Edit the color and size of the headlines and icons.
  • Style the lists with margin, padding and border.
  • If the button classes where applied correctly in the HTML part, all buttons should be already styled. Create an active class to highlight currently selected buttons.
  • Add a :hover class to the city select list item using the image draggable.png inside the img folder.

Arrange sections

  • Use z - index to arrange all sections. The element with the highest z-index is always in front of other elements.

CSS Effects (D5)

  • Add a fade overlay that covers the underlying content, when the sidebar is open.Use a width of 100% for the sidebar-fade-overlay and add a dark half-transparent background-color.
  • Create a half-transparent background gradient for the table.
  • Add shadow to your button style
  • Use a div element and CSS animation to create an animated loading indicator.

CSS Transform & Transitions (D6)

  • Provide a background-color transition for the mainview element when the color is changed. Test the transition with your browser developer tools by changing the background-color manually.
  • Use two css-rotated spans with white background - color and a height of 3px, to build an X-icon for the close-sidebar button

F: JavaScript

What the app will do

For better understanding, compare with our best practice weather app

.
  • The app shows the weather data of the selected day. The user can flick through seven consecutive days using the next and previous button
  • For today, tomorrow and the day after tomorrow detail information will be generated. The detail information consists of hourly weather data
  • The sidebar menu allows you to select different cities
  • One can add additional cities
  • The cities can be sorted
  • You can add a new city based on your GPS location using the Locate me button
  • In the menu settings you can choose between different units for temperature and windspeed
  • We will use real weather data from an Internet Service
  • While the data is loaded, a loading animation is shown. Sidebar and detail view are animated.
  • The background color changes depending on current temperature

  • Go to body and add an onload = "alert('Hello World');" attribute. Reload the page and your message will be displayed.

Variables (F3)

G: jQuery

jQuery Basics (G1)

  • Embed the jQuery library into your index.html – either offline or via jQuery CDN
  • Replace document.getElementById() with jQuery's query selector $('#id')
  • Hint: Whenever you want to access a DOM element, be sure the DOM has already loaded! Put this code always into the $(document).ready() event handler function!
  • Instead of style.display, now use jQuery's$.show(), .$.hide() or $.toggle() to control visibility of your detailsview. Try different animation speeds
  • Clean up index.html: Move all javascript code to your external script file. Only use script tags in the head to embed jQuery and your JS code. Remove all onclick attributes in your HTML and use .click() instead.

jQuery Effects (G2)

  • Fade in the animated loading indicator
  • Call $.fadeOut() after 500 ms
  • Hint: Check the documentation how the jQuery animation queue works
  • Add an animation that smoothly displays the detailsview, when button-details is clicked

jQuery & HTML (G3)

  • Replace all innerHTML manipulations with jQuery's html() function
  • Use jQuery's $.each() to create DOM elements for each element in your cities array. Then $.append() them to your city-list
  • Instead of creating your details table out of HTML strings, use jQuery's element creation, like $.("div")
  • When the user adds a new city, append it to the city-list in the sidebar
  • Track the current unit of wind speed and temperature with a global variable.Use jQuery's $.click() on the sidebar unit chooser buttons to change the current unit.
  • Modify your conversion functions so that it considers the current unit

jQuery Functions (G4)

  • Create an object conversion with your conversion methods getWindspeed and getTemperature
  • Use $.each() and $.extend() to merge the conversion object into every weather data object
  • Change updateWeatherMainHTML() so that it uses the conversion methods to get the converted values for temperature and wind speed

jQuery UI (G5)
