Introduction

This is the practical task assignment for the Interaction Programming Block Course (introductory slides). The block course is taught at the Institute of Ergonomics of the Technische Universität München in the context of master courses in Human Factors Engineering, Mechanical Engineering, and Computer Sciences in the . This document is authored by Markus Zimmermann, David Schopf, Niklas Lütteken, and Annika Stumpf.

The exercises aim to learn the web technologies HTML5/CSS3/JavaScript by developing a weather application during the course.

D Style and Layout

Global Stylesheet (D1)

Add the same external stylesheet main.css to all .html files. Always try to use appropriate CSS selectors that only affect those elements, which are supposed to be affected. Too imprecise selectors may cause disarrangement of your elements. If necessary, improve your html structure by adding new class or id attributes. Apply the generic rule:

  • Assign ids for content which is present once (menu, current temperature, current weather, …)
  • Assign classes for elements, which are present multiple times (e.g. multiple rows of detail information, multiple buttons, …)

Layout your Views (D2/D3)

Transform your app to into a single page application

  • Instead of using multiple HTML files (one for each view) we will now use only one page for all views. Copy and paste your detailview into your index.html just below mainview. Now detailview should overlap mainview.
  • Use the display property for showing and hiding sidebar and detailview. Try the developer console to test very quickly: Add and change CSS properties directly in Chrome (Elements > Styles).

Main view

  • Use fixed heights for the mainview element (e.g. 600px), header, datepicker (both e.g. 40px) and footer (60px). Apply a width of 100% to all elements.
  • Use relative positioning for all buttons and absolute positioning for the headlines inside header and datepicker.
  • Use a fixed width and height for buttons in header and datepicker.
  • Use relative positioning for all elements inside content. Set appropriate widths and center all elements or rather group of elements.
  • Use absolute positioning to pin the footer to the bottom. Set a fixed height and full width to the footer button.

Detail view

  • Use same heights and widths for the detailview elements as used before for the mainview. Set a fixed height for the content (height detailview – height header) and make it scrollable by using the property overflow: auto.
  • Use absolute positioning for the header to pin it to the top.
  • Set a fixed height for all table rows and divide the table rows into columns using percental widths.

Sidebar

  • Use same heights for sidebar as used before.
  • Use a width of 70% for the sidebar and a dark background-color.

Add style information (D4)

MainView

  • Add a dark, half-transparent background to your button class, mainview-datepicker and mainview-footer.
  • Create :hover and :active styles for all buttons to highlight the user interaction.
  • Use font properties to style color and font-size of the icons and appearance of headlines.
  • Use padding & margin to set enough space between the elements.

DetailView

  • Use the CSS selector :nth-child() to set a half-transparent darker background-color to odd rows and lighter background-color to even rows.
  • If the button classes where applied correctly in the HTML part, the close button and header should be already styled.

Sidebar

  • Edit the color and size of the headlines and icons.
  • Style the lists with margins, paddings and borders.
  • If the button classes where applied correctly in the HTML part, all buttons should be already styled. Create an active class to highlight currently selected buttons.
  • Add a :hover class to the city select list item using the image draggable.png inside the img folder.

Final positioning

This step hides all sections except the mainview. Make sure that all CSS Styling is completed before you proceed.
  • Instead of display we will now use position: absolute for showing and hiding all sections and overlays:
  • Wrap your view sections mainview and detailview into a full-size wrapper block element with overflow:hidden; to prevent your views from being displayed.
  • Move the sidebar out of sight by using left
  • Do the same for the detailview with top
  • Use z-index to arrange all sections.

          #wrapper {
            width: 100%;
            height: 100%;
            overflow: hidden;
          }

          #mainview {
              width: 100%;
              height:100%;
              position: absolute;
              top: 0;
              left: 0;
              z-index: 1;
          }

          #detailview {
            width: 100%;
            height: 100%;
            position: absolute;
            top: 100%;
            left: 0;
            z-index: 20;
          }

          #sidebar {
              width: 70%;
              height: 100%;
              position: absolute;
              left: -70%;
              z-index: 10;
          }
